## 这是什么

智能制造工程（专业代码 080213T）是 2017-2018 年教育部新批的机械类本科专业。它位于**机械工程 × 控制科学 × 计算机科学**的交叉地带，目标是培养能设计、集成、优化智能产线和数字化工厂的复合型工程师。

计算机专业有 [developer-roadmap](https://github.com/kamranahmedse/developer-roadmap)（30万+ ⭐），前端/后端/DevOps 各自有清晰的学习路径。但智能制造工程——**没有**。这个 repo 就是为了补上这个空缺。

### 这份路线图包含

- ✅ 按 8 个学期的完整课程地图（含学分、优先级标注、前置依赖）
- ✅ **T 型能力模型**：控制/自动化纵深 + 机械基础够用 + 软件/数据决定天花板
- ✅ **每学期项目里程碑**：每个学期应该做出什么东西
- ✅ 全部先修/后修课程依赖链（可视化 ASCII 图）
- ✅ **学科竞赛指南**：西门子杯、工训大赛、挑战杯等含金量排序
- ✅ **证书体系**：1+X 证书、PLC 工程师认证等
- ✅ 考研（控制科学与工程 081100）vs 就业（智能工厂方向）双线规划
- ✅ S5（大三上）决断标准：走考研还是走就业
- ✅ B站/教材/在线课程/硬件/GitHub 仓库 资源索引
- ✅ 毕业时应该能做什么——可勾选的技能清单

### 适合谁看

- 智能制造工程大一/大二在读学生

## 🎯 T 型能力模型

智能制造工程师的核心竞争力是 **T 型结构**——不是三轴平均用力，而是**一横两竖**：

```
                     🟨 软件/数据能力
                     │  (决定你能走多高)
                     │
    ─────────────────┼─────────────────  🟦 机械基础 (横杠：够用即可)
                     │
                     │
                     🟩 控制/自动化能力
                        (纵深：你的饭碗)
```

### T 型详解

| 层级 | 领域 | 策略 | 核心课程 | 毕业时应该能 | 对应岗位 |
|------|------|------|---------|-------------|---------|
| **横杠** | 🟦 机械工程 | **够用即可**（75+就行，不跟机械专业卷画图） | 工程图学 1/2、工程力学、机械原理与设计、机械材料与制造、数控技术 | 看懂零件图/装配图，理解公差配合，知道常见传动和加工工艺 | 工艺工程师、机械设计助理 |
| **竖杠 1** | 🟩 控制/自动化 | **拼命扎深**（85+目标，这是你的饭碗） | 电工技术 → 电子技术 → 自动控制原理 → 单片机 → PLC → 传感检测 | 设计顺序控制逻辑，读写梯形图，选传感器型号，排查电路信号，独立接线 | PLC工程师、自动化工程师、设备调试 |
| **竖杠 2** | 🟨 计算机/数据 | **决定天花板**（能写代码≠能当码农，是用代码解决工业问题） | Python → 数据库 → 工业互联网 → 数字孪生 → 机器学习入门 | 写脚本处理生产数据，理解 MES 数据流，用 Git，了解 OPC UA/MQTT | MES实施顾问、数字化工程师、数据分析工程师 |

大三结束:  机械 ██████░░░░  60%  "够用了"

> **关键认知**：你不是要跟机械专业比画图、跟计算机专业比刷题、跟自动化专业比控制理论。你的价值在于**能把三者串起来**——看懂机械结构后设计控制方案，写程序把设备和系统连起来。这是纯机械/纯计算机/纯自动化的人都做不到的。

### 智能制造 vs 相近专业：你到底强在哪

> 这一节是给你面试时用的。当面试官问「你这专业跟自动化有什么区别」，你不需要临场组织语言。

| 对比维度 | 🟦 机械工程 | 🟩 自动化 | 🟨 计算机科学 | 🏭 **智能制造工程（你）** |
|------|-----------|---------|------------|----------------------|
| **核心能力** | 机械结构设计与力学分析 | 控制理论与系统建模 | 算法与软件架构 | **系统集成：把设备、控制、数据串起来** |
| **对机械的理解** | ⭐⭐⭐⭐⭐ 精通 | ⭐⭐ 基本不懂 | ⭐ 完全不懂 | ⭐⭐⭐ 能看图、懂工艺、知道设备怎么动 |
| **对控制的理解** | ⭐ 基本不懂 | ⭐⭐⭐⭐⭐ 精通 | ⭐ 完全不懂 | ⭐⭐⭐⭐ **PLC+传感器+驱动，能独立做控制柜** |
| **对编程的理解** | ⭐ 可能不会 | ⭐⭐ 会写 MATLAB | ⭐⭐⭐⭐⭐ 精通 | ⭐⭐⭐⭐ **Python+SQL+IIoT，能写工业数据处理和通信** |
| **看一条产线时看到的是** | 齿轮、轴承、公差 | 传递函数、稳定性裕度 | 代码架构、数据库表 | **物料流 × 信息流 × 控制流，三个流怎么对接** |
| **典型工作产出** | 零件图、装配图、工艺卡 | 控制模型、Simulink 仿真 | 软件系统、API、算法 | **跑着代码的控制柜 + 能讲清楚为什么这样设计** |
| **职业天花板** | 高级机械工程师 / 技术总监 | 控制系统架构师 | 软件架构师 / CTO | **智能工厂解决方案架构师——三个方向都懂才能做的位置** |

**一句话回怼质疑**：「机械工程师知道设备怎么造，但不知道控制系统怎么写。自动化工程师知道控制算法怎么推，但没摸过 MES 数据库。我的工作是把他们的产出集成到一个产线里跑起来——他们各自只懂一块，我三块都懂。」

> ⚠️ **但这个定位有一个前提**：你必须在控制/自动化方向扎得足够深（T 型的竖杠够长），否则就会变成「三块都只懂皮毛」。这就是为什么本路线图把控制/自动化标为 🔴 第一优先级。

| 自动控制原理 < 80，或补修拖后腿，或经济压力变大 | → **就业为主**（路线B），考研试水或放弃 |
| 中间地带（75-84） | → **双线继续**，S6 再决断 |

**数学类课均分** = (高数A1 + 高数A2 + 线代B + 概率论 + 复变函数) / 5

> ⚠️ **S5 结束前不报名考研。** 成绩出来后（约2月初）再决定报不报。

## ✅ 毕业技能清单

> 毕业时逐条打勾。这些是你面试时能拿出手的东西。

### 🟦 机械（够用即可）

- [ ] 能看懂零件图和装配图，理解三视图投影原理
- [ ] 理解公差配合（基孔制/基轴制、公差等级）
- [ ] 知道常见传动机构：齿轮、链、带、丝杠、连杆、凸轮
- [ ] 了解常见加工工艺：车铣刨磨钻 + 数控加工
- [ ] 能用 SolidWorks 做简单零件建模和装配

### 🟩 控制/自动化（饭碗——拼命扎深）

- [ ] 会用万用表、示波器排查电路信号
- [ ] 能独立接一个简单电气控制柜（PLC + 继电器 + 接触器 + 传感器 + 急停）
- [ ] 梯形图基本指令 + 功能指令熟练，能写顺序控制/报警处理/手动自动切换/定时器+计数器逻辑
- [ ] 理解 PID 控制原理，能用 Simulink 调参仿真，能在 STM32 上实现数字 PID
- [ ] 了解伺服驱动接线与控制（脉冲+方向 / 模拟量）
- [ ] 了解 HMI/触摸屏组态基本操作（至少在博途上做过一个画面：按钮+指示灯+数值显示）
- [ ] 了解工业通信协议概念：Modbus RTU/TCP、OPC UA、MQTT、Profinet
- [ ] 能根据现场需求选传感器型号（接近/光电/温度/压力/编码器）

### 🟨 嵌入式/编程（决定天花板）

- [ ] STM32 GPIO/Timer/PWM/中断/ADC/串口/I2C/SPI 全部用过
- [ ] 独立完成过一个带闭环控制（PID）的 STM32 项目
- [ ] 能用 Python 写数据处理脚本（pandas + matplotlib）
- [ ] 会用 Git/GitHub 管理代码，有自己的项目仓库
- [ ] 了解数据库基础（会用 SQL 写查询，了解时序数据库概念）
- [ ] 了解 Docker 基本使用（工业互联网方向加分项）

### 🔧 系统/集成

- [ ] 理解 MES/ERP 的数据流：工单→报工→质检→入库
- [ ] 了解数字孪生基本概念和技术栈（数据采集→建模→仿真→可视化）
- [ ] 能用 Plant Simulation 或类似工具做一个简单产线仿真
- [ ] 了解智能制造典型场景（参考工信部《智能制造典型场景参考指引》）
- [ ] 了解 OEE（设备综合效率）计算方法和改进思路

### 🤖 机器人/视觉（加分项）

- [ ] 了解工业机器人基本坐标系（基座标/工具坐标/用户坐标）
- [ ] 知道示教器基本操作（至少一个品牌：ABB/KUKA/FANUC）
- [ ] 了解机器视觉基本流程：采集→预处理→特征提取→检测/识别
- [ ] 用过 OpenCV 做简单图像处理（边缘检测/模板匹配）

### 🗣 软技能

- [ ] 能用「背景→问题→方案→结果」结构写技术文档
- [ ] 能对着电路图/梯形图/代码讲清楚自己的项目
- [ ] CET-4 必须过，CET-6 尽量过
- [ ] 面试时能说清楚「智能制造是什么、我能在里面做什么」
- [ ] 会用 AI 工具（ChatGPT/Claude）辅助学习和开发
