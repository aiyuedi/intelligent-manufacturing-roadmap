智能制造工程（专业代码 080213T）是 2017-2018 年教育部新批的机械类本科专业。它位于**机械工程 × 控制科学 × 计算机科学**的交叉地带，目标是培养能设计、集成、优化智能产线和数字化工厂的复合型工程师。

计算机专业有 [developer-roadmap](https://github.com/kamranahmedse/developer-roadmap)（30万+ ⭐），前端/后端/DevOps 各自有清晰的学习路径。但智能制造工程——**没有**。这个 repo 就是为了补上这个空缺。

## 🎯 T 型能力模型

智能制造工程师的核心竞争力是 **T 型结构**——不是三轴平均用力，而是**一横两竖**：

```
                     🟨 软件/数据能力
                     │  (决定你能走多高)
                     │
    ─────────────────┼─────────────────  🟦 机械基础 (横杠：够用即可)
                     │
                     │
                     🟩 控制/自动化能力
                        (纵深：你的饭碗)
```

### T 型详解

| 层级 | 领域 | 策略 | 核心课程 | 毕业时应该能 | 对应岗位 |
|------|------|------|---------|-------------|---------|
| **横杠** | 🟦 机械工程 | **够用即可**（75+就行，不跟机械专业卷画图） | 工程图学 1/2、工程力学、机械原理与设计、机械材料与制造、数控技术 | 看懂零件图/装配图，理解公差配合，知道常见传动和加工工艺 | 工艺工程师、机械设计助理 |
| **竖杠 1** | 🟩 控制/自动化 | **拼命扎深**（85+目标，这是你的饭碗） | 电工技术 → 电子技术 → 自动控制原理 → 单片机 → PLC → 传感检测 | 设计顺序控制逻辑，读写梯形图，选传感器型号，排查电路信号，独立接线 | PLC工程师、自动化工程师、设备调试 |
| **竖杠 2** | 🟨 计算机/数据 | **决定天花板**（能写代码≠能当码农，是用代码解决工业问题） | Python → 数据库 → 工业互联网 → 数字孪生 → 机器学习入门 | 写脚本处理生产数据，理解 MES 数据流，用 Git，了解 OPC UA/MQTT | MES实施顾问、数字化工程师、数据分析工程师 |

> **关键认知**：你不是要跟机械专业比画图、跟计算机专业比刷题、跟自动化专业比控制理论。你的价值在于**能把三者串起来**——看懂机械结构后设计控制方案，写程序把设备和系统连起来。这是纯机械/纯计算机/纯自动化的人都做不到的。

### 智能制造 vs 相近专业：你到底强在哪

> 这一节是给你面试时用的。当面试官问「你这专业跟自动化有什么区别」，你不需要临场组织语言。

| 对比维度 | 机械工程 | 自动化 | 计算机科学 | **智能制造工程（你）** |
|------|---------|------|----------|----------------------|
| **核心能力** | 结构设计与力学分析 | 控制理论与系统建模 | 算法与软件架构 | **系统集成：把设备、控制、数据串起来** |
| **对机械/控制/编程的理解** | ⭐⭐⭐⭐⭐ / ⭐ / ⭐ | ⭐ / ⭐⭐⭐⭐⭐ / ⭐⭐ | ⭐ / ⭐ / ⭐⭐⭐⭐⭐ | ⭐⭐⭐ / ⭐⭐⭐⭐ / ⭐⭐⭐⭐ |
| **看一条产线的视角** | 齿轮、轴承、公差 | 传递函数、稳定性裕度 | 代码架构、数据库表 | **物料流 × 信息流 × 控制流，三个流怎么对接** |
| **工作产出** | 图纸、工艺卡 | Simulink 模型 | 软件系统、API | **跑着代码的控制柜 + 能讲清楚为什么这样设计** |
| **职业天花板** | 高级工程师 / 总监 | 控制系统架构师 | 软件架构师 / CTO | **智能工厂解决方案架构师** |

**一句话回怼质疑**：「机械工程师知道设备怎么造，但不知道控制系统怎么写。自动化工程师知道控制算法怎么推，但没摸过 MES 数据库。我的工作是把他们的产出集成到一个产线里跑起来——他们各自只懂一块，我三块都懂。」

> ⚠️ **但这个定位有一个前提**：你必须在控制/自动化方向扎得足够深（T 型的竖杠够长），否则就会变成「三块都只懂皮毛」。这就是为什么本路线图把控制/自动化标为 🔴 第一优先级。

## 🧭 普通人最优路径

> **这条路线的前提**：你从大一就在智能制造工程专业，不是转专业进来的。你均分 80-85，英语一般（高考 110±），不是竞赛保送生，有正常的社交娱乐需求。这完全不丢人——大多数本科生本来就是这样。
>
> 网上那些「大一拿国奖、大二发 SCI、大三保研、大四 BAT SSP offer」的路线的评论区是沉默的大多数。本章不是让你选「最好的路」，而是帮你找到**确定性最高、容错空间最大**的路。本章基于桂电 2025 级培养方案的真实课程时序和学分负载写就，但分析方法适用于任何院校的智能制造工程专业。

### 你是普通人，不是废物

先看一个「普通学生」在智能制造工程的真实处境——以桂电培养方案为模板，我们逐学期算账：

```
S1（大一上）— 27.25 学分，5 门硬课：
  高数 A1(5.5) + Python(3) + 工程图学 1(2.5) + 专业认知实习(1) + 电子认知实习(1)
  刚进大学，还在适应节奏。如果高数 A1 考了 80-85，正常。

S2（大一下）— 29.25 学分，整个大学负荷最重的学期：
  高数 A2(5.5) + 大物 B(4) + 线代 B(2) + 工程图学 2(2.5) + 物理实验 B(1.5)
  + 计算机绘图训练(1) + 工程制图测绘(1) + CET-4
  同时对付 7 门理工课。每门都交作业，每周都在赶 ddl。
  普通人在这种密度下能全科 75+ 已经是胜利。

S3（大二上）— 25.75 学分，数学收尾 + 工程入门：
  复变(2) + 概率(3) + 电工(1.5) + 工程力学(4) + 智能制造概论(2) + 金工实习 4 周(4)
  电工技术是整个控制/自动化线的起点——从这里开始分岔。

S4（大二下）— 25.75 学分，机械+电子双线并进：
  电子技术(3.5) + 电工电子实验(1) + 机械原理(3) + 机械材料(3) + 两个课设(2.5)
  电子技术和机械原理同时上，一个偏电一个偏机，脑子要切频道。

S5（大三上）— 22.25 学分，专业核心爆发：
  自动控制原理(2) + 单片机(2+1.5) + PLC(2) + 传感(2) + 数控(2)
  + IIoT(2) + 计算机网络(2) + 电子工程训练(1.5)
  一学期 8 门硬核专业课。自动控制原理一门课同时决定考研和就业两条路。
  
  ⚠️ 这是你整个大学压力最大的学期，没有之一。
```

> 如果你逐行读下来感觉「这就是我」，那接下来要谈的不是你能不能逆袭，而是**怎么在这条路上少踩坑、不翻车**。

### 🔴 路线 A 的真相：不是不可能，是代价很大

桂电 081100 控制科学与工程，B 区国家线（总分 250，政/英≥31，数一/专业课≥47）。分数线看起来很低对吧？但分数线低 ≠ 容易考。

**考研不是看分数线，是看你排第几名。** 25 个名额，报 200 人，你只需要跑赢 175 个人。看起来不难。但你的竞争对手里至少有 30 个二战考生——他们去年已经刷了一遍数学一。

**数学一的真实要求**（不是分数线，是「能考到那个分的底子」）：

| 你的底子 | 数一预估 | 说明 |
|---|---|---|
| 高数 A1 80-85 + 高数 A2 80-85 | 100-110/150 | 刚好过线，不高 |
| 线代 85+ | 线代部分稳 | 你的潜在优势 |
| 概率（S3 学） | 取决于 S3 概率期末多少分 | 概率 80+ → 数一概率部分 80% |
| 复变（S3 学） | 拉普拉斯变换是自控的前置，不是数一重点 | 会算就行 |

> 数学一的区分度不在高数上册（大家都刷过），在**高数下册的级数和曲面积分**。S2 高数 A2 期末考 80 分和 95 分的人，到数一真题里差距会再放大。

**英语一的真实要求：**

英语一阅读理解的文章从《经济学人》《自然》《科学》《卫报》直接摘——不是改写，是原文。没有 CET-6 的阅读速度，第一次做真题时你会发现一句话读三遍还把握不住意思。这不是你不努力，是大学英语 1-4 和英语一之间的鸿沟真实存在。

**考研时间账（逐科拆开，不按「感觉」算）：**

```
数学一基础轮 — 高数上(80h) + 高数下(80h) + 线代(40h) + 概率(40h) = 240h
数学一强化轮 — 1000 题/660 题                                ≈ 200h
数学一冲刺   — 真题 15 年 + 模拟卷 10 套                      ≈ 80h
808 自控     — 胡寿松课本精读+课后题+桂电真题 10 年           ≈ 200h
英语一       — 单词 5500(100h) + 阅读 30 年真题(80h)
               + 翻译(20h) + 作文模板(20h)                    = 220h
政治         — 1000 题 + 肖四肖八 + 腿姐背诵手册              ≈ 120h
────────────────────────────────────────────────────────
总计 ≈ 1060 小时净学习时间
```

1060h ÷ (S5-S6-S7 三个学期共 15 个月 × 30 天) ≈ **2.4h/天**。看起来能做到。

但真正的困难不是平均，是**峰值冲突**：

> S5 大三上期末。周一到周五：单片机课设要验收、PLC 梯形图要交、传感实验报告要补。周六：自动控制原理期末考。周日：你想刷一套数一真题，发现脑子已经转不动了。

普通人会在这种冲突里内耗——考研复习进度落后 → 焦虑 → 课内效率下降 → 更焦虑。这才是考研失败的真实原因。不是不够聪明，是精力分配不过来。

**结论**：你能考上吗？能。但失败概率 > 50%。考研失败 = 错过秋招黄金窗口 + 作品集没打磨 + 只能春招捡剩饭。

### 🟢 路线 B 的确定性

珠三角制造业（深圳、广州、东莞、佛山）招本科生的真实逻辑排序：

```
1. 会不会 PLC / 能不能独立接线  ← 面试直接给你看梯形图，让你讲
2. 有没有实习经历               ← 哪怕只是暑假去车间待了两个月
3. 能不能讲清楚自己做过的项目    ← 代码+视频+技术文档 > 绩点
4. CET-4 过了没                 ← 外企 CET-6 是硬门槛，民企不卡
5. 绩点                         ← 排在最后，只要不挂科就行
```

这些东西**不依赖一场考试**。你花 200 小时做 STM32 闭环控制项目，做完就是你的，面试能讲。考研花 500 小时刷数一，考砸了归零。

**代价**：起薪 6-10K vs 研究生 12-18K；大厂研发岗基本没有；5 年内薪资靠跳槽补。

### 🟡 路线 C：对大多数人最优

**核心思想**：以就业为底线，把考研从「孤注一掷」降级为「看 S5 成绩再说的选项」。

#### 为什么这个策略适合大多数人

**① S1-S4 的时间比你想象的值钱**

从大一到大三上，你有约 6000 小时自由时间（160 周 × 15h 自由时间 + 3 个寒暑假）。如果把这 6000h 砸在数学一上，你会错过：

| 投入 | 产出 | 考研失败后这个产出还在吗？ |
|---|---|---|
| Arduino 入门项目（40h） | 第一个硬件作品 + GitHub 仓库 | ✅ 永久 |
| PLC 梯形图从入门到能写顺序控制（80h） | 博途项目文件 + 录屏 | ✅ 永久 |
| STM32 闭环温控系统（100h） | GitHub 仓库 + 演示视频 + 技术文档 | ✅ 永久 |
| 暑假实习（8 周） | 简历第一行工作经验 + 人脉 | ✅ 永久 |
| 数学一第一轮（240h） | 真题正确率从 30% 提到 50% | ❌ 考砸归零 |

> 考研时间不是免费的。每 1 小时砸在数学一上，就是 1 小时没有砸在 PLC 或 STM32 上。

**② 数学一不需要提前三年开始**

自动控制原理 S5 才上。你 S1 就开始刷数一，等于在**不知道专业课能不能考好**的情况下先砸了 500 小时。S5 自动控制原理考了 75，你发现 808 自控是你的短板——前两年的数学一白刷了。

反过来：S1-S4 把就业底线筑牢，S5 自动控制原理学好了→再做决断。这时候你已经有 Arduino+PLC+STM32 三个项目 + 一段实习在手里。考不上不慌，考上就赚。

**③ 这个专业的培养方案本身就偏向就业**

培养方案的实践教学占 36 学分（21.2%），比专业理论课（19.5 学分，11.5%）多一倍。这个专业的设计意图就不是培养纯理论研究者——是要培养能在产线上干活的人。

### 🎯 最终推荐

**走路线 C。** 用 S1 到大三上这 2.5 年把就业底线筑牢——PLC 能写、STM32 能调、至少有一段实习经历。S5 期末看「自动控制原理」的成绩再决定考不考研。

**这不是退缩。** 这是把 2.5 年的时间押在**你最能控制的事情**上。PLC 技能 + 项目经历 + 实习经历不会因为一张考研试卷就归零。而数学一和英语一——你 S5 再押也不迟。

**唯一的前提**：这 2.5 年别摸鱼。Arduino 五个实验要跑完，PLC 梯形图要能独立写顺序控制，STM32 要做出一个带闭环控制的项目，暑假要想办法弄一段实习。如果这四件事你都没做——那路线 A/B/C 都救不了你。

**但正常走这条路的普通学生，毕业时手里会有**：
- 🟢 Arduino 入门项目（GitHub 仓库 + 视频）

- 🟡 （如果考研成功）桂电 081100 录取通知书

**Sem5 核心任务**：
- 🔴 **自动控制原理冲 85+**，这是最高优先级——同时决定考研和就业

- PLC 梯形图学到能独立写顺序控制+报警逻辑
- 记录自动控制原理每次考试成绩，作为 S5 末决断的核心数据

**🏗️ 项目里程碑**（这个学期的项目直接放进求职作品集）：
> **STM32 闭环电机控制系统**：编码器反馈 + PID 速度/位置控制 + OLED 实时显示 + 按键设定目标值。这就是你简历上的第一个核心项目。

## 🔗 先修后修关键链

### 数学链
```
高等数学A1(S1) → 高等数学A2(S2) → 复变函数与积分变换(S3) → 工程计算方法(S4)
线性代数B(S2) → 概率论与数理统计(S3) ────────────┘
线性代数B(S2) → 自动控制原理(S5) [状态空间需要矩阵运算]
```

### 物理·电工·电子链
```
大学物理B(S2) → 电工技术(S3) → 电子技术(S4) → 单片机原理与应用(S5)
                                          └→ 机电传动控制与PLC(S5)
电子认知实习(S1) → 物理实验B(S2) → 电工与电子技术实验(S4) → 电子工程训练1(S5)
```

### 图学·制造·设计链
```
工程图学1(S1) → 工程图学2(S2) → 工程力学(S3) → 机械原理与设计基础(S4)
                                                 → 数控技术与机床(S5)
计算机绘图训练(S2) → 机械材料与制造基础(S4)
机械原理与设计基础(S4) → 课程设计(S4)
```

### 编程·数据·物联网链
```
Python语言程序设计(S1) → 概率论与数理统计(S3) → 大数据与云计算技术(S4)
                                                  → 工业互联网与物联网技术(S5)
```

### 🟢 智能工厂模块链（主线）
```
电工技术(S3) → 电子技术(S4) → 机电传动控制与PLC(S5) ─┐
自动控制原理(S5) ─────────────────────────────────┼→ 智能生产计划管理MES/ERP(S6)
传感与检测技术(S5) ────────────────────────────────┘          │
                                                              ↓
                                              智能工厂综合实训(S7)
                                                      │
                                                      ↓
                                              毕业设计(S8)
```

### 竞赛参与策略

| 年级 | 策略 |
|------|------|
| **大一** | 不参赛，先打基础。关注学校机器人社团/创新实验室，围观学长比赛。 |
| **大二** | 尝试参赛。从工训赛、电赛或数学建模起步，积累经验。**
- 大二暑假是比赛集中期**，提前半年组队 |
| **大三** | **冲刺。西门子杯是核心目标**。这时候你有自控原理+PLC+单片机，刚好全部用上。拿奖对考研复试和秋招都是巨大加分 |
| **大四** | 秋招/考研为主。有余力可以再打一次西门子杯做毕业设计素材 |

> 💡 **关键建议**：竞赛不在于多，在于**精**。选 1-2 个认真做，比报了 5 个都划水强。西门子杯 + 工训赛是最佳组合——前者偏软件和系统集成，后者偏机械和加工。

## 📡 工业通信协议动手实验

> Modbus 是这个行业最古老的通用语言。大部分 PLC、传感器、变频器都支持它。花一个下午跑通一次，比读十篇文章有用。

### Modbus RTU 零成本实验（Arduino + RS485）

**硬件**：Arduino Uno × 2 + MAX485 模块 × 2 + 杜邦线（¥60 以内搞定）

**实验步骤**：
1. 主机 Arduino：用 [SimpleModbusMaster](https://github.com/angeloc/simple-modbus) 库发送读取请求
2. 从机 Arduino：用 [SimpleModbusSlave](https://github.com/angeloc/simple-modbus) 库响应，返回虚拟温度值
3. 主机 OLED 显示"从机温度：26.5°C"
4. 拔掉一根线——看通信超时怎么处理

**产出**：一个视频 + 代码仓库，面试时说「我用 Arduino 跑通过 Modbus RTU 主从通信」

### 协议扩展（做了 Modbus RTU 之后，有余力再碰）

| 协议 | 动手方式 | 什么时候做 |
|------|---------|-----------|
| Modbus TCP | Node-RED 连接 Modbus 模拟器 | S4-S5 |
| MQTT | ESP32 + MQTT Explorer + Node-RED 仪表盘 | S5（跟上 IIoT 课） |
| OPC UA | [open62541](https://github.com/open62541) + Python 客户端 | S7（数字孪生方向） |
| CAN Bus | 两个 STM32 + MCP2515 CAN 模块互发消息 | S5-S6（汽车/机器人方向） |

## 📊 行业现状：2025-2026 你毕业时的市场

> 这一节帮你建立「我学这些东西到底在市场上值多少钱」的认知。数据来源：猎聘 2025 报告、工信部/人社部人才缺口数据、2025-2026 招聘 JD 采样。

### 人才缺口和薪资

| 数据项 | 数值 | 来源 |
|---|---|---|
| 智能制造人才缺口 | **~450 万人** | 工信部/人社部 |
| 高档数控机床和机器人缺口 | ~520 万人 | 人社部 |
| 智能制造领域平均年薪 | **28.6 万元**（含硕士/经验岗） | 猎聘 2025 |
| 本科应届月薪（珠三角） | 6K-10K | BOSS 直聘 2025 采样 |
| PLC 工程师 3 年经验供需比 | **1:6**（6 个岗位抢 1 人） | 行业共识 |

### 2025-2026 招聘 JD 高频技能词

| 频次 | 技能 | 你的培养方案覆盖度 |
|---|---|---|
| 极高（80%+ JD） | PLC、AutoCAD/SolidWorks、MES/ERP、SQL | PLC ✅ | MES/ERP 🟡 | SQL ❌ |
| 高（60-80% JD） | Python、SCADA/HMI、工业机器人、精益生产 | Python 🟡 | SCADA ❌ | 机器人 🟡 |
| 中（40-60% JD） | OPC UA/Modbus/Profinet、Power BI/Tableau、数字孪生、FMEA | 协议概念 ✅ | BI ❌ | 数字孪生 🟡 |
| 新兴（热） | AI/ML 质量预测、边缘计算、5G+工业互联网、具身智能 | 几乎全 ❌（课内不教，需要课外补齐） |

### 核心发现

**① 你不需要全会**。"极高"和"高"频次的技能，PLC + SQL + Python 三条腿站稳就超过 80% 的竞争者。

**② 工业 AI 是最大变量**。2025 年工业企业 AI 应用占比从 10% 跃升到 47.5%，华为盘古大模型在一个工厂跑 1600+ 个视觉质检点。你不需做算法，但要知道 AI 在产线上怎么用——这个认知差就是面试时的区分度。

**③ 证书不如项目**。1+X 证书有含金量，但面试官更在意"你真的跑过 Modbus 通信吗"而不是"你考了 PLC 应用编程证书"。先把项目做出来，再顺手考证。

**④ 桂电在珠三角的校友网络是隐形优势**。深圳制造业招聘 35.2%、广州 31.8% 集中在珠三角。你的学校在这个区域有认知度——善用。
