# 🤝 贡献指南

感谢你考虑为这个项目做出贡献！智能制造工程是一个新兴专业，社区的力量能帮助更多同学少走弯路。

## 如何贡献

### Level 1：修正错误
发现课程代码不对？薪资数据过时了？复试线变了？软件版本更新了？
→ **直接提 PR**，在 PR 描述中附上正确的来源链接。

### Level 2：补充内容
你是智能制造工程的学生/毕业生，有自己的经验总结？
→ **先开 Issue 讨论**，描述你想补充什么内容，为什么需要它，然后再提 PR。

### Level 3：新增章节
你觉得应该加一整章新内容（比如某个方向的深度指南、某个地区就业分析）？
→ **先开 Issue 用结构化大纲描述**，等社区讨论达成共识后再动手。

### Level 4：翻译
你愿意把本路线图翻译成其他语言？
→ **开 Issue 说明你想翻译的目标语言**，等确认后翻译到独立文件（如 `README-en.md`）。

## 内容规范

- **数据必须有来源**：薪资、分数线、课程代码等请附上官方链接或多个交叉验证源
- **资源优先选免费**：推荐的学习资源优先选免费、在线可访问的（B站、YouTube、开源项目）
- **不要 GeeksforGeeks 链接**（内容质量低、广告多）
- **一个资源链接 + 一行说明**，不要堆砌链接不解释为什么推荐它
- **中性表述**：路线图中的「推荐/可选/不推荐」必须注明理由，不凭个人喜好

## Issue 模板

提交 Issue 时请包含以下信息：

```
**类型**：[数据修正 / 内容补充 / 章节建议 / 翻译 / 其他]
**位置**：指出是在 README 的哪个章节
**描述**：清楚描述你的问题或建议
**来源**：如果是数据修正，附上来源链接
```

## PR 规范

- PR 标题用中文或英文都可以，但要描述清楚改了什么
- 一个 PR 只做一件事（修一个数据、加一个小节、改一个章节）
- 大改动先开 Issue 再动手
- 保持与现有风格一致（emoji 导航、表格格式、ASCII 图对齐）

## 行为准则

- 尊重不同背景、不同院校的贡献者
- 讨论聚焦内容质量，不人身攻击
- 欢迎反对意见——附上理由和来源就行
